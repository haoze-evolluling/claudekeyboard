# Dark Mode Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add dark mode support to SyncTouch using M3 `darkColorScheme` with system-follow default and manual settings override.

**Architecture:** Android `values-night/` resource qualifier provides dark color roles and theme overrides. `AppCompatDelegate.setDefaultNightMode()` controls mode at runtime. Settings page `MaterialButtonToggleGroup` offers Follow System / Light / Dark.

**Tech Stack:** Kotlin, Android Views + XML, Material Design 3, `AppCompatDelegate`

## Global Constraints

- Drawable XML colors (`home_card`, `home_hero_*`, `home_soft_panel`, etc.) remain unchanged — only M3 color roles switch
- No new Kotlin classes or files — only modify existing `MainActivity.kt`
- All user-facing strings added to `strings.xml` in Chinese
- Follow existing settings page UI patterns: `MaterialCardView` with `cardCornerRadius="22dp"`, `cardBackgroundColor="@color/home_card"`, `strokeColor="@color/home_card_stroke"`
- `SharedPreferences` key: `settings_prefs`, `theme_mode` key with values `MODE_SYSTEM` / `MODE_LIGHT` / `MODE_DARK`

### Task 5: Add theme logic in MainActivity

**Files:**
- Modify: `app/src/main/java/com/haoze/claudekeyboard/MainActivity.kt`

**Interfaces:**
- Consumes: `toggle_theme_mode`, `btn_theme_system`, `btn_theme_light`, `btn_theme_dark` from layout (Task 4)
- Produces: Theme mode read from `SharedPreferences` on startup, applied via `AppCompatDelegate.setDefaultNightMode()`; toggle group listener writes preference and applies mode immediately

- [ ] **Step 1: Add import for `MaterialButtonToggleGroup`**

Add this import near the top of `MainActivity.kt` (after the existing `com.google.android.material` imports):

```kotlin
import com.google.android.material.button.MaterialButtonToggleGroup
```

- [ ] **Step 2: Apply theme mode in `onCreate()` before `setContentView`**

In `MainActivity.kt`, in the `onCreate()` method, add after `enableEdgeToEdge()` (line 81) and before `setContentView(R.layout.activity_main)` (line 82):

```kotlin
        val prefs = getSharedPreferences("settings_prefs", Context.MODE_PRIVATE)
        val themeMode = prefs.getString("theme_mode", "MODE_SYSTEM") ?: "MODE_SYSTEM"
        AppCompatDelegate.setDefaultNightMode(
            when (themeMode) {
                "MODE_LIGHT" -> AppCompatDelegate.MODE_NIGHT_NO
                "MODE_DARK" -> AppCompatDelegate.MODE_NIGHT_YES
                else -> AppCompatDelegate.MODE_NIGHT_FOLLOW_SYSTEM
            }
        )
```

Note: the existing `val prefs` in `setupSettingsPage()` on line 405 will shadow this variable — that's fine, they're in different scopes.

- [ ] **Step 3: Add theme toggle logic in `setupSettingsPage()`**

In `MainActivity.kt`, inside `setupSettingsPage()`, add after the existing slider setup code (after line 428):

```kotlin
        val toggleTheme = findViewById<MaterialButtonToggleGroup>(R.id.toggle_theme_mode)
        val themeMode = prefs.getString("theme_mode", "MODE_SYSTEM") ?: "MODE_SYSTEM"
        toggleTheme.check(
            when (themeMode) {
                "MODE_LIGHT" -> R.id.btn_theme_light
                "MODE_DARK" -> R.id.btn_theme_dark
                else -> R.id.btn_theme_system
            }
        )
        toggleTheme.addOnButtonCheckedListener { _, checkedId, isChecked ->
            if (!isChecked) return@addOnButtonCheckedListener
            val mode = when (checkedId) {
                R.id.btn_theme_light -> "MODE_LIGHT"
                R.id.btn_theme_dark -> "MODE_DARK"
                else -> "MODE_SYSTEM"
            }
            prefs.edit().putString("theme_mode", mode).apply()
            AppCompatDelegate.setDefaultNightMode(
                when (mode) {
                    "MODE_LIGHT" -> AppCompatDelegate.MODE_NIGHT_NO
                    "MODE_DARK" -> AppCompatDelegate.MODE_NIGHT_YES
                    else -> AppCompatDelegate.MODE_NIGHT_FOLLOW_SYSTEM
                }
            )
        }
```
